## 🎯 各数据集完整推荐配置与调参逻辑

### 1️⃣ ML-100K（长序列 + 相对密集）

#### 数据特点
- **特点**：长历史信息充足，模型容易学到用户偏好，但也可能过拟合。RL微调可以更激进，直接对齐指标。

#### 推荐配置

```yaml
# ========== 位置感知模块参数 ==========
use_position_aware: true
position_importance_mode: 'hybrid'           # 混合模式最优
importance_supervision_weight: 0.08          # 中等强度
scheduler_temperature: 1.5                   # 较低温度

# ========== 排序损失参数 ==========
pref_loss_scale: 1.8                         # 排序损失权重较高


```

#### 调参逻辑
- **位置模块**：长序列对位置的依赖相对复杂，`hybrid`模式效果好。温度`scheduler_temperature`不宜过高。
- **排序损失**：数据稠密，排序信号强，`pref_loss_scale`可以设得较高。

---

### 2️⃣ Baby（小规模 + 短序列 + 高稀疏）

#### 数据特点
- **特点**：信息极其稀缺，位置先验至关重要。RL微调需要非常保守，防止破坏已学到的脆弱结构。

#### 推荐配置

```yaml
# ========== 位置感知模块参数 ==========
use_position_aware: true
position_importance_mode: 'position_based'   # 简单线性更稳定
importance_supervision_weight: 0.03          # 较低，避免过度约束
scheduler_temperature: 2.5                   # 较高温度

# ========== 排序损失参数 ==========
pref_loss_scale: 1.7                         # 排序损失权重中等偏高


#### 调参逻辑
- **位置模块**：短序列依赖简单的位置衰减，`position_based`最有效。高稀疏需要高`scheduler_temperature`来保证采样覆盖。
- **排序损失**：`pref_loss_scale`适中，补充信号但不过分主导。

---

### 3️⃣ Beauty（小-中规模 + 短序列 + 高稀疏）

#### 数据特点
- **特点**：物品空间比Baby大，稀疏度更高，冷启动问题突出。调参策略与Baby类似，但需要更注重几何正则。

#### 推荐配置

```yaml
# ========== 位置感知模块参数 ==========
use_position_aware: true
position_importance_mode: 'position_based'   # 同Baby
importance_supervision_weight: 0.04          # 略高于Baby
scheduler_temperature: 2.3                   # 略低于Baby

# ========== 排序损失参数 ==========
pref_loss_scale: 1.0                         # 排序损失权重较低，防止干扰主任务

```

#### 调参逻辑
- **位置模块 & 排序损失**：逻辑同Baby，但由于更稀疏，`pref_loss_scale`更保守。
---

### 4️⃣ Toys（中等规模 + 短序列 + 极端稀疏）

#### 数据特点
- **特点**：极端稀疏，物品多，长尾问题严重。所有调参都必须以“稳定”为第一要务。

#### 推荐配置

use_position_aware: true
position_importance_mode: 'position_based'   # 保持简单
importance_supervision_weight: 0.035         # 较低
scheduler_temperature: 2.8                   # 较高

# ========== 排序损失参数 ==========
pref_loss_scale: 1.0                         # 排序损失权重保守

#### 调参逻辑

---

### 5️⃣ Sports（中-大规模 + 短序列 + 高稀疏）

#### 数据特点
- **特点**：规模较大，数据量比前几个Amazon子集多，可以支撑更复杂的学习模式。

#### 推荐配置

```yaml
# ========== 位置感知模块参数 ==========
use_position_aware: true
position_importance_mode: 'learned'          # 尝试学习模式
importance_supervision_weight: 0.045         # 中等
scheduler_temperature: 2.2                   # 略低

# ========== 排序损失参数 ==========
pref_loss_scale: 1.2                         # 排序损失权重中等

# ========== Stage4 RL 微调参数 ==========
rl_enable: true
rl_epochs: 3
rl_lr_scale: 0.08                            # 学习率可以适度放开
rl_kl_beta: 0.01                             # KL约束可以回到标准值
rl_geom_alpha: 0.1                           # 几何约束标准值
```

#### 调参逻辑
- **位置模块**：数据量足够，可以尝试`learned`模式。

---

### 6️⃣ Yelp（超大规模 + 中等序列 + 极端稀疏）

#### 数据特点
- **特点**：规模极大，稀疏度也极高。虽然数据多，但有效信号少。RL微调风险与机遇并存。

#### 推荐配置

```yaml
# ========== 位置感知模块参数 ==========
use_position_aware: true
position_importance_mode: 'learned'          # 大规模数据适合学习
importance_supervision_weight: 0.12          # 较高
scheduler_temperature: 3.0                   # 最高

# ========== 排序损失参数 ==========
pref_loss_scale: 0.5                         # 排序损失权重保守

```

#### 调参逻辑
- **位置模块**：大数据量适合`learned`模式，且监督权重`importance_supervision_weight`可以给高一些。

## 🔧 通用调参流程（含Stage4）

#### 阶段1：验证基线（1-2次运行）
- `use_position_aware: false`, `pref_loss_scale: 0.0`, `rl_enable: false`。
- 记录基线性能。

#### 阶段2：位置模块调参（3-5次运行）
- `use_position_aware: true`, `pref_loss_scale: 0.0`, `rl_enable: false`。
- 根据数据集特点，搜索`scheduler_temperature`和`importance_supervision_weight`。

#### 阶段3：排序损失调参（4-8次运行）
- 固定阶段2最优参数，`rl_enable: false`。
- 搜索`pref_loss_scale`。

#### 阶段4：RL微调调参（3-5次运行）
- 固定前三阶段最优模型和参数，`rl_enable: true`。
- **优先调 `rl_lr_scale` 和 `rl_kl_beta`**，这是决定微调力度和稳定性的核心。
- 其次微调`rl_geom_alpha`，平衡排名和几何。
- `rl_epochs`通常固定为2-3即可。

## 💡 最后的建议

1.  **从推荐值开始**：新指南为你提供了覆盖三个模块的完整起点。
2.  **分阶段调参**：严格按照“基线 → 位置 → 排序 → RL”的顺序，每一步都在上一部最优的基础上进行。
3.  **RL调参要保守**：尤其是在稀疏数据集上，RL是“锦上添花”而非“推倒重来”。小的学习率和强的KL约束是保证稳定的关键。
4.  **监控RL过程**：日志中会打印`[Stage4] Epoch X: avg_reward=...`，观察平均奖励是否为正且稳定。
